# OpenRouter Free Models Layer

[中文说明](./README.zh-CN.md)

A small data pipeline that tracks OpenRouter free models and publishes a stable, app-facing `model_layer.json`.

The repository is designed for consumers that want a predictable routing layer instead of hard-coding volatile OpenRouter model IDs. It refreshes the free-model inventory, derives a few stable profile aliases, and stores daily snapshots for comparison.

## What This Repository Publishes

- `model_layer.json`: the main app-facing artifact, with stable profile aliases and normalized model metadata.
- `known_free_models.json`: the latest full baseline of detected free models.
- `daily_snapshots/YYYY-MM-DD.json`: point-in-time snapshots for historical diffing.
- `model_changes.json`: a run-level diff artifact generated by the script for local inspection.

## Why Use It

- Free-model detection is strict: every field in `pricing` must be zero.
- App integrations can route through stable profile aliases instead of provider-specific model IDs.
- Ordered fallback candidates are generated for each profile.
- Daily snapshots make model churn easy to audit.
- Capability ranking can improve candidate order when an OpenRouter API key is available.

## Profiles

The generated layer currently exposes three built-in profiles:

- `chat.default.free`: general-purpose free chat.
- `chat.reasoning.free`: reasoning-focused free chat.
- `chat.longctx.free`: long-context free chat.

Each profile uses `selection: "ordered-fallback"` and includes a `candidate_model_ids` array ordered from most preferred to least preferred.

## Quick Start

### Consume the published model layer

Use the raw GitHub artifact directly:

```text
https://raw.githubusercontent.com/cyclez2000/openrouter-free-models/main/model_layer.json
```

Recommended client behavior:

- Fetch once per day.
- Cache locally for 24 hours.
- Keep the previous cache if refresh fails.
- Route by profile alias first, then fall back through `candidate_model_ids`.

### Run locally

```bash
pip install -r requirements.txt
python check_models.py
```

Optional PowerShell example with API key:

```powershell
$env:OPENROUTER_API_KEY="your_key"
python check_models.py
```

## Environment Variables

- `OPENROUTER_API_KEY`: optional for the model list request, recommended for capability ranking.
- `OPENROUTER_CAPABILITY_RANKING`: set to `false` to disable LLM-based candidate reordering.
- `OPENROUTER_CAPABILITY_RANKER_MODEL`: overrides the default ranker model.

Default ranker model:

```text
openai/gpt-oss-120b:free
```

If the ranker is disabled or the API key is missing, the script falls back to local heuristic ordering and still completes successfully.

## Data Contract

Top-level keys in `model_layer.json`:

- `schema_version`
- `updated_at`
- `source`
- `stats`
- `profiles`
- `models`

Example profile entry:

```json
{
  "chat.default.free": {
    "description": "General-purpose free chat profile.",
    "selection": "ordered-fallback",
    "candidate_model_ids": [
      "model-a:free",
      "model-b:free"
    ]
  }
}
```

Example model entry:

```json
{
  "id": "google/gemma-3-27b-it:free",
  "name": "Google: Gemma 3 27B (free)",
  "context_length": 131072,
  "input_modalities": ["image", "text"],
  "output_modalities": ["text"],
  "is_moderated": false,
  "tags": ["long_context", "reasoning", "text", "vision"]
}
```

## Update Flow

GitHub Actions currently runs the pipeline:

- on a daily schedule
- on manual dispatch
- on pushes that change `check_models.py` or the workflow itself

During a run, the workflow:

1. Fetches the current OpenRouter model catalog.
2. Filters strictly free models.
3. Writes the latest baseline, daily snapshot, and model layer.
4. Opens a GitHub issue when added or removed free models are detected.
5. Commits updated tracked artifacts back to `main`.

## Notes

- `model_layer.json` is the stable integration target for apps.
- `known_free_models.json` retains richer raw metadata from OpenRouter.
- `model_changes.json` is generated by the script but is not part of the workflow's committed artifact set.
